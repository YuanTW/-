# 量化交易儀表板 — 完整部署教學

## 專案結構

```
quant_dashboard/
├── app.py                    ← Streamlit 主儀表板
├── config.py                 ← 設定管理
├── database.py               ← 資料庫操作
├── runner.py                 ← 策略執行引擎
├── telegram_notifier.py      ← Telegram 推播
├── scheduler_manager.py      ← APScheduler 排程
├── strategies_config.json    ← 策略清單設定（你要編輯這個）
├── strategies/
│   └── strategy_template.py ← 策略腳本範本（把你的腳本改成這個格式）
├── outputs/                  ← 自動產生，存放圖表與結果
├── requirements.txt
├── Procfile
├── railway.toml
└── .env.example
```

## STEP 1 — 修改你的策略腳本

你現有的腳本只需要在**最後**加上幾行，把結果存到正確位置：

```python
# 在你的策略腳本最後加上這段
import os, json
from pathlib import Path

output_dir = Path(os.environ.get("STRATEGY_OUTPUT_DIR", "./output"))
output_dir.mkdir(parents=True, exist_ok=True)

# 1. 推薦方向（文字）
(output_dir / "recommendation.txt").write_text("做多", encoding="utf-8")

# 2. 損益圖（你原本就有 fig，直接存）
fig.savefig(output_dir / "chart.png", bbox_inches="tight")

# 3. 選填：訊號代碼（BUY / SELL / HOLD）
(output_dir / "result.json").write_text(
    json.dumps({"signal": "BUY", "details": "你的說明"}, ensure_ascii=False),
    encoding="utf-8"
)
```

> **重要**：腳本最上方加上 `import matplotlib; matplotlib.use("Agg")`，
> 雲端沒有螢幕，這行可避免錯誤。

## STEP 2 — 設定 strategies_config.json

編輯 `strategies_config.json`，把你的每個策略腳本填入：

```json
{
  "strategies": [
    {
      "id": "tw_trend",
      "name": "台股趨勢追蹤",
      "script": "strategies/tw_trend.py",
      "schedule": "0 8 * * 1-5",
      "enabled": true,
      "description": "每個交易日 08:00 執行"
    },
    {
      "id": "us_momentum",
      "name": "美股動量策略",
      "script": "strategies/us_momentum.py",
      "schedule": "30 22 * * 2",
      "enabled": true,
      "description": "每週二 22:30 執行"
    }
  ]
}
```

**Cron 格式說明（分 時 日 月 星期）：**

| 範例 | 說明 |
|------|------|
| `0 8 * * 1-5` | 週一到週五 08:00 |
| `30 8 * * 1-5` | 週一到週五 08:30 |
| `0 22 * * 3` | 每週三 22:00 |
| `0 8,14 * * 1-5` | 週一到週五 08:00 和 14:00 |
| `0 */4 * * *` | 每 4 小時 |

## STEP 3 — 設定 Telegram Bot

### 取得 Bot Token
1. 在 Telegram 搜尋 `@BotFather`
2. 發送 `/newbot`，依指示設定 Bot 名稱
3. BotFather 會給你一串 Token，格式：`123456789:ABCdefGhIJKlmNopQRstuVwXyz`

### 取得你的 Chat ID
1. 對你剛建立的 Bot 發任意一則訊息（如：`/start`）
2. 在瀏覽器打開：
   ```
   https://api.telegram.org/bot你的TOKEN/getUpdates
   ```
3. 在回傳的 JSON 中找到 `"chat": {"id": 數字}`，那個數字就是你的 Chat ID

## STEP 5 — 部署到 Railway

1. 前往 [railway.app](https://railway.app)，用 GitHub 帳號登入
2. 點 **New Project** → **Deploy from GitHub repo**
3. 選擇你剛建立的 repository
4. Railway 會自動偵測並開始部署

### 新增 PostgreSQL 資料庫
1. 在 Railway 專案頁點 **+ New** → **Database** → **PostgreSQL**
2. Railway 會自動把 `DATABASE_URL` 環境變數注入你的服務，不需手動設定

### 設定環境變數
在 Railway 的服務頁面，點 **Variables** 分頁，新增：

```
TELEGRAM_BOT_TOKEN    = 你的 Bot Token
TELEGRAM_CHAT_ID      = 你的 Chat ID
DASHBOARD_TITLE       = 量化交易儀表板
TIMEZONE              = Asia/Taipei
```

### 完成部署
1. Railway 部署完成後會給你一個網址，格式：`https://xxx.railway.app`
2. 點進去就是你的儀表板

## 常見問題

**Q：Railway 免費嗎？**
A：有免費額度（每月 $5 credit），足夠跑這個儀表板。若策略頻繁執行可考慮升級 $5/月方案。

**Q：如何新增更多策略？**
A：把你的腳本放入 `strategies/` 資料夾，在 `strategies_config.json` 新增一筆設定即可，不需修改任何其他程式碼。

**Q：圖片在雲端重啟後消失？**
A：Railway 的磁碟是暫時性的，重啟後 `outputs/` 資料夾會清空，但資料庫的文字結果（推薦方向）不會消失。若要永久保留圖片，可以在策略腳本中加上上傳到 Imgur 或 Cloudinary 的邏輯。

**Q：Telegram 沒收到通知？**
A：先確認你有對 Bot 發過訊息（Bot 不能主動對沒有互動過的使用者發訊息），再確認 Chat ID 是數字格式。

**Q：排程時區是哪裡？**
A：預設是台北時間（Asia/Taipei），可在環境變數 `TIMEZONE` 更改。
